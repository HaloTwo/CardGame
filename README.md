# Burger Monster Card Battle

## 1. 프로젝트 개요

Unity로 제작한 세로형 턴제 카드 배틀 게임 프로토타입입니다.

플레이어와 상대는 각각 6장의 카드를 가지고 전투를 시작합니다.  
각 진영은 3장의 카드를 전장에 공개 배치하고, 나머지 카드는 뒤집힌 대기 카드로 보관합니다.

플레이어는 자신의 턴에 아군 카드를 선택한 뒤 기본공격 또는 카드효과를 사용합니다.  
상대방의 모든 카드를 제거하면 승리하고, 플레이어의 모든 카드가 제거되면 패배합니다.

## 5. 조작 방법

1. 로비 화면에서 `전투 시작` 버튼을 누릅니다.
2. 전투 화면에서 사용할 아군 카드를 선택합니다.
3. `기본공격` 또는 `카드효과` 버튼을 선택합니다.
4. 공격 대상이 될 적 카드를 선택합니다.
5. 효과가 적용된 뒤 상대 턴으로 전환됩니다.
6. 상대 턴은 AI가 자동으로 진행합니다.
7. 한쪽의 모든 카드가 제거되면 승리 또는 패배 결과 화면이 표시됩니다.

## 6. 구현 기능 목록

### 전투 시스템

- 턴 진행 시스템
- 카드 선택 시스템
- 기본공격 처리
- 카드별 스킬 처리
- HP 계산
- 반격 피해 처리
- 카드 제거 처리
- 빈 슬롯 발생 시 대기 카드 자동 배치
- 승리 및 패배 판정
- 간단한 상대 AI 자동 턴 진행

### 카드 타입

- 일반
  - 자신의 현재 HP만큼 대상에게 피해를 주고, 대상의 현재 HP만큼 반격 피해를 받습니다.
- 원거리
  - 자신의 현재 HP만큼 대상에게 피해를 주고, 반격 피해를 받지 않습니다.
- 무쌍
  - 대상에게 자신의 현재 HP만큼 피해를 주고, 인접한 적 카드 중 1장에게 추가 피해를 줍니다.
- 힐러
  - 자신의 턴 시작 시 자신을 제외한 아군 카드들의 HP를 회복합니다.
- 폭탄
  - 대상에게 피해를 주고, 나머지 적 카드에게 추가 광역 피해를 줍니다.
- 흡혈
  - 대상에게 피해를 준 뒤 자신의 HP를 회복합니다.
- 광전사
  - 잃은 HP만큼 추가 피해를 주고 자신도 피해를 받습니다.
- 수호자
  - 낮은 피해를 주는 대신 자신의 HP를 회복합니다.
- 관통
  - 대상과 양옆 인접 카드에게 피해를 줍니다.

### UI / UX

- 세로형 전투 화면
- 현재 턴 표시
- 카드 HP 표시
- 카드 타입 표시

- 기본공격 / 카드효과 버튼
- 대기 카드 수 표시
- 승리 / 패배 결과 화면
- 공격 카드 포커스 연출
- 피격 카드 흔들림 연출
- 피해량 텍스트 연출
- 화면 플래시 및 HIT 텍스트 연출
- 신규 카드 배치 시 카드 뒤집기 연출

### 로비 기능

- 전투 시작
- 덱 편집

## 7. 주요 코드 구조

### Assets/3.Script/Core

- `Singleton.cs`
  - 씬 단위 싱글톤 베이스 클래스입니다.

### Assets/3.Script/CardBattle/Core

- `CardBattleManager.cs`
  - 전투 시작, 턴 전환, 플레이어 입력 흐름, 상대 AI 턴 진행, 결과 처리, UI 이벤트 연결을 담당합니다.

- `BattleField.cs`
  - 각 진영의 전장 슬롯 3개와 대기 카드 목록을 관리합니다.
  - 카드 제거 후 빈 슬롯이 생기면 대기 카드에서 자동으로 새 카드를 배치합니다.

### Assets/3.Script/CardBattle/Rules

- `CardBattleRules.cs`
  - 기본공격, 카드별 스킬, HP 계산, 반격, 힐러 회복 효과 등 전투 규칙을 처리합니다.

- `EnemyBattleAI.cs`
  - 상대 턴에서 사용할 카드와 공격 대상을 선택하는 간단한 AI 로직을 담당합니다.

### Assets/3.Script/CardBattle/Data

- `BattleCardDefinition.cs`
  - 카드 이름, 타입, HP, 이미지, 설명을 설정할 수 있는 ScriptableObject 데이터입니다.

- `BattleCardData.cs`
  - 전투 중 사용하는 카드의 고정 데이터입니다.

- `BattleCardRuntime.cs`
  - 전투 중 변하는 카드 상태를 관리합니다.
  - 현재 HP, 소유자, 슬롯 인덱스, 사망 여부를 포함합니다.

- `BattleDeckFactory.cs`
  - 기본 덱 또는 저장된 플레이어 덱을 전투용 카드 목록으로 생성합니다.

- `CardCatalog.cs`
  - 도감, 덱 편집, 성장에서 사용하는 카드 목록 데이터를 관리합니다.

- `CardPlayerProfile.cs`
  - PlayerPrefs를 이용해 전적, 보유 재화, 카드 레벨, 덱 편성을 저장합니다.

### Assets/3.Script/UI

- `CardBattleView.cs`
  - 전투 화면 UI 참조와 텍스트, 버튼, 결과창 갱신을 담당합니다.

- `CardSlotView.cs`
  - 카드 슬롯 표시, 선택 강조, 피격 연출, 피해량 텍스트, 카드 뒤집기 연출을 담당합니다.

- `CardBattleRuntimeViewBuilder.cs`
  - 전투 씬에 UI 참조가 없을 경우 실행 가능한 기본 전투 UI를 런타임에 생성합니다.

- `StartSceneController.cs`
  - 로비 버튼, 덱 편집, 카드 도감, 성장 패널, 전투 씬 이동을 담당합니다.

- `StartSceneRuntimeViewBuilder.cs`
  - 로비 씬에 UI 참조가 없을 경우 실행 가능한 기본 로비 UI를 런타임에 생성합니다.

### Assets/3.Script/Editor

- `CardBattleSceneBuilder.cs`
  - 전투 씬 오브젝트와 기본 카드 데이터를 생성하는 에디터용 빌더입니다.

## 8. 구현 시 신경 쓴 부분

### 1. 전투 로직과 UI 분리

전투 규칙은 `CardBattleRules`에서 처리하고, 전장 상태는 `BattleField`, 카드 상태는 `BattleCardRuntime`, UI 표시는 `CardSlotView`와 `CardBattleView`로 분리했습니다.

### 2. 카드별 효과 확장성

카드 타입에 따라 다른 효과를 적용할 수 있도록 `BattleCardType`과 `CardBattleRules`를 중심으로 구조를 구성했습니다.  
새 카드 타입을 추가할 경우 카드 타입 enum과 룰 처리 함수를 추가하는 방식으로 확장할 수 있습니다.

### 3. 대기 카드 자동 배치

전장 카드가 제거되면 `BattleField`에서 빈 슬롯을 확인하고, 대기 카드가 남아 있으면 자동으로 새 카드를 배치하도록 구현했습니다.

### 4. 프로토타입 재현성

씬에 UI가 정상적으로 연결되어 있지 않아도 테스트가 가능하도록 런타임 UI 빌더를 구현했습니다.  
이를 통해 제출된 코드만으로도 기본 전투 화면과 로비 화면을 재구성할 수 있도록 했습니다.
